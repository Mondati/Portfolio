03. Proyectos

Proyecto destacado

API Tienda Electrónica

Backend para una tienda de electrodomésticos que necesitaba escalar sus operaciones de productos, carritos y ventas de forma independiente. Diseñé una arquitectura de microservicios con 7 servicios distribuidos: cada uno tiene su propia base de datos MySQL, se registra en Eureka automáticamente, recibe configuración desde un Config Server centralizado en Git, y se comunica a través de un API Gateway único. Docker Compose orquesta el entorno completo con un solo comando. Implementé Circuit Breaker con Resilience4j para manejar fallos entre servicios sin propagar errores en cascada.
  • Spring Cloud 2023
  • Docker Compose
  • Eureka
  • OpenFeign
  • Resilience4j
  • MySQL 8

Proyecto final · Certified Tech Developer

DH Booking Reservas

Plataforma de reservas gastronómicas que resuelve el problema de gestionar disponibilidad: un usuario no puede reservar un horario que ya está ocupado. La validación de solapamiento de fechas se implementó con una query nativa SQL que detecta conflictos directamente en base de datos, evitando colisiones en carga concurrente. El acceso está protegido con Spring Security y dos roles diferenciados (ADMIN / USER), con contraseñas encriptadas con BCrypt.
  • Spring Security
  • BCrypt
  • MySQL 8
  • Query nativa SQL
  • CORS

Proyecto destacado

API Clínica Odontológica

API REST para digitalizar la gestión de una clínica: registro de pacientes, altas y bajas de odontólogos, y asignación de turnos con validaciones de integridad. El acceso está controlado con Spring Security y dos roles (ADMIN / USER): solo el administrador puede modificar datos sensibles. Incluye documentación interactiva con Swagger UI y un frontend en HTML/CSS/JS que consume la API directamente.
  • Spring Boot 2.7
  • Spring Security
  • BCrypt
  • H2
  • Swagger / OpenAPI

Proyecto universitario · UDA

Blackjack

Juego de Blackjack de escritorio desarrollado como proyecto final de la materia Laboratorio 2 en la Universidad del Aconcagua. El desafío principal era separar correctamente la lógica del juego de la interfaz gráfica. Apliqué el patrón MVC con Java Swing, implementé un dealer con reglas estándar (stand-on-17), caché de imágenes para optimizar la carga de recursos y persistencia del historial de partidas en disco. Incluye suite de tests con JUnit 5.
  • Java 21
  • Java Swing
  • MVC
  • JUnit 5
  • Maven
